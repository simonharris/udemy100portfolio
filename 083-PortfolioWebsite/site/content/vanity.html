As the name suggests, django-vanityurls was originally designed to support vanity URLs---short, memorable routes to content deeper within your site, but it has proven to be more generally applicable to any internal redirects.

It's a Django middleware that jumps in after routing has happened, and only does its stuff if routing generates a 404. If the requested URL is in the database, it issues the redirect with the HTTP response code of your choice.

You can install it via pip install vanity-urls, and everything you need is online:

  • Source code on Github
  • Official project page on PyPI
  • Documentation on Read the Docs

It's only a tiny project, but I've found it useful, and it's in use behind Music Live COLCHESTER. As a smaller project I've taken the opportunity to be a bit relentless with the professional practices, so for example there is 100% test coverage, and the code lints to 10/10.
