The original Wiki, developed by Ward Cunningham in the mid 90s, was a quantum leap, both technologically and socially: a key step in democratisation of the web. Previously, to contribute to the web, one had to be a computer scientist in a university or research institute somewhere, with a server under your desk.

Of course, since the only people with web access in those days still were computer scientists, the content was somewhat niche, and grew into a great resource for aspiring software engineers like me.

This is my tribute to it, and an experiment in how easy it would be to recreate the Wiki using modern technologies like Python and Django. It's all up there on the web itself:

  • Source code on Github
  • Official project page on PyPI
  • Documentation on Read the Docs

I'm at a crossroads with the project. Should it stay as close as possible to the spirit of the original, or should it embrace the things modern users expect from such as system, like rich text editing and social features? Rest assured I won't be replicating the egregious single-page interface Ward inexplicably grafted onto it in recent years.

I'll revisit it one day, but in the meantime I use it myself, and find it pretty useful.
