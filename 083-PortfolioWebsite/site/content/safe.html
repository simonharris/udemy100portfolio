Safe Cracker is a numeric logic puzzle published in one of the Sunday papers here in the UK. It's kind of fun to tackle, and after solving it on paper a bunch of times, I got into programming the solutions in Prolog, because why not.

Things kind of escalated from there, and now the system can solve the puzzle just from a cameraphone photograph of the puzzle's English description. It's a whole Rube Goldberg-like contraption comprising three subsystems:

  • A Natural Language Processing and logical reasoning engine written in Prolog, which parses the puzzle text using a Definite Clause Grammar, and converts the clues into CLP(FD) constraints
  • A back-end system built on the Flask framework in Python, which performs the Optical Character Recognition, interfaces with Prolog, and exposes a streaming HTTP API
  • A responsive, interactive front-end JavaScript web app build with Vue.js

All the code is open source and available via the safecracker.pl repository on Github, and there's a longer discussion of the project on the site's about Safe Cracker page. If you don't have a copy of the specific paper to hand, there are a few example puzzles stored within the site, so you can watch it work.
